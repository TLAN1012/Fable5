## 緣起

起心動念,是一位朋友做了一張**關原之戰的 3D 圖**。藍醫師看了心癢,想說自己也來做做看——
於是找 CMini(Claude **Fable 5**)從一場長津湖開始。結果越做越有興趣,索性一路做下去:
**兩場韓戰、三場二戰、一場上古**,從零下四十度的蓋馬高原,做到黃昏沙塵的亞述平原,
橫跨兩千三百年的六場會戰,在一個下午之內全部完成並公開上線。

這個專案同時也是一次實驗:驗證 Fable 5 模型**自行完成完整工作流**的能力——
從史料梳理、地理建模、程式撰寫、音效合成,到 GitHub 部署與上線驗證,全程自主執行,
人只負責選題、驗收與說「OK,推上去」。

## 戰史回顧 — 六役一線牽

六場會戰各自獨立,放在一起卻互為鏡像:

- **長津湖(1950)與砥平里(1951)** 是同一場戰爭的一體兩面。長津湖是聯軍被志願軍的
  夜襲與人海分割包圍、靠空投鋼橋與海運才得以全身而退;僅僅兩個半月後的砥平里,
  聯軍學會了答案——**環形防禦+空中補給+裝甲馳援**,5,600 人頂住 2.5 萬人三夜圍攻,
  人海戰術自此失靈。兩集連看,正好是「問題」與「解法」。
- **法國戰役(1940)與市場花園(1944)** 則是裝甲突穿的正反面教材。1940 年德軍七個
  裝甲師穿越阿登、十日到海,證明「集中裝甲走敵人認為不可能的路」可以亡人之國;
  1944 年盟軍想用同一邏輯反攻——沿一條公路直插萊茵河——卻敗在情報輕忽、時間表僵硬
  與單線補給:鐮刀成功的每一個前提,在安恆正好全部缺席。**同一把刀,兩種命運。**
- **台兒莊(1938)** 是六役中唯一以血肉之軀對抗機械化軍隊而獲勝的一場。它的勝利方程式
  與砥平里異曲同工——死守(池峰城)+外線機動(湯恩伯)——只是把空投與戰車,
  換成了川軍的土砲與 57 把大刀。
- **高加米拉(前331)** 放在最後,卻是一切的源頭:斜行誘敵、拉開缺口、預備隊回身、
  直取敵軍重心——亞歷山大在兩千三百年前示範的機動戰原理,在阿登的裝甲楔形、
  湯恩伯的迂迴、乃至砥平里的內外夾擊裡,一再重演。**武器會變,原理不變。**

> 各集面板內均註明範圍。

## 做法 — 技術架構

每一集都是**單一 `index.html` 檔**,零建置流程、零後端、零外部素材,開檔即玩:

- **3D 引擎**:Three.js 0.160(CDN importmap),`OrbitControls` 自由視角
- **地形**:程序化生成——以層疊正弦雜訊做山脊/丘陵,解析函式 `terrainH(x,z)` 同時供
  網格頂點與所有物件貼地使用;河流/海岸/湖泊用有號距離場下挖,頂點著色畫出
  雪原、圩田、麥田、沙漠四種風土
- **軍標**:藍/紅圓柱(上古篇改橫隊方塊),CSS2DRenderer 掛中文標籤(部隊・指揮官・兵力);
  位置由 keyframe 陣列 `[[t,x,z],...]` 線性插值,**一切皆為時間 t 的純函數**,
  時間軸隨意拖動、倒帶皆成立
- **時間軸**:1 個 slider 統治一切——事件字幕、晝夜循環、天候(降雪/降雨/沙塵)、
  箭頭顯隱、軌跡點亮、單位覆滅變灰/登船淡出,全部由 t 驅動
- **特效**:單張 radial-gradient canvas 貼圖打天下——爆炸、火場、濃煙、照明彈、
  傘降、沙塵、白刃閃光全是它的 Sprite 變體;另有傘降鋼橋、鐮刀戰車、戰象、
  船團、機群等專屬模型
- **音效**:Web Audio **即時合成,無任何音檔**——暴風雪/砲擊低頻震波/斯圖卡耶利哥號角/
  衝鋒號/法軍手搖警報器/船舶汽笛/薩爾平克斯軍號/戰鼓/萬人吶喊/馬蹄/兵刃相擊,
  全部用振盪器+噪音+濾波器調出來,並依鏡頭距離衰減
- **態勢小視窗**(本系列招牌):SVG 手繪戰略圖,在關鍵時段自動浮現——
  蘆屋基地空運航線、英荷雙空降航線、徐州會戰南北對進、曼斯坦鐮刀對比、亞歷山大東征路線
- **無障礙**:A−/A＋ 全域字級縮放(CSS 變數,localStorage 記憶)、音效開關、0.5–4× 調速

## 做法 — 工作流(Fable 5 自主執行)

每一集的生產線固定為六步,模型自行完成:

1. **史實梳理**:時間軸、戰鬥序列、指揮官、兵力、名言、爭議數字標註範圍
2. **地理建模**:把真實地形抽象成座標系(1 單位 ≈ 0.1–8 km 依戰役尺度),定 keyframe
3. **單檔實作**:複用上一集模板,替換地形/部隊/事件/特效/音效套件
4. **語法驗證**:`node --check` 擷取模組碼靜態檢查
5. **本機驗收**:開瀏覽器給藍醫師過目,聽取「OK/調整」
6. **部署上線**:push GitHub → Pages 自動部署 → `curl` 輪詢驗證 HTTP 200 → 更新首頁目錄

第一集從零寫起;之後每一集只改「資料與風土」,架構不動——
這正是越做越快、一個下午完成六集的原因。
